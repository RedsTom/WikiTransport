# Language persistence fix

Summary: Language no longer resets to English when navigating between pages

Date: 2026-05-16

- Fixed language resetting to English when clicking internal links (e.g., going from projects to the editor)
- Switched from URL-prefix-based locale detection (`/fr/...`) to cookie-only strategy, eliminating reliance on URL prefixes
- Replaced `<a data-sveltekit-reload>` language switchers with `<button onclick={() => setLocale(l)}>` for instant, no-reload switching
- Added `{#key useLocale()}` in root layout to re-render all translated content on locale change
